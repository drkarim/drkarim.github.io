Issue

Utility.php is in .git ignore, but it is still not ignored by GIT and tracked actively. Utility has production and development server addresses hard-coded, and thus the versions on PROD and DEV servers should not be synced.

Changes in PROD

If the file is changed in PROD, the changes need to be made on the PROD file itself, using an SSH editor such as PICO.

Once the changes are committed, the DEV needs to PULL these changes, however, a backup of utility on DEV is made before the PULL. This is just keeping GIT tracking of the file at bay, thinking that it is being synced. Steps:

On Production server: 

1. pico utility.php (Make changes) on PROD 
2. git add utility.php 
3. git commit

On Development server: IMPORTANT: Make a backup of development version of utility 

git checkout -- utility.php 
git pull

Changes in DEV

The changes to the file in development does not need to be committed. It always stays in the stack.
