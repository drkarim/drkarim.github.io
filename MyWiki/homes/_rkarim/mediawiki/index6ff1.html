Branching

Branches are best created with source-tree. Once you create a branch and write code, move files from unstage to stage. Once satisfied commit changes to current branch. Put commit message. So steps:

  • Unstage to stage
  • Commit to current branch

File can be ignore by adding to .gitignore: 

# My ignore rules #
# See: https://gist.github.com/octocat/9257657 #
/cgi-bin/utility.php
/admin/invoices_html
/admin/adhoc_invoices_html

Merging

The best way to merge are these steps. Also look at this link on stack overflow

  • Commit all changes to branch
  • git checkout master - switch to master branch
  • Merge from source tree

Rebase

To integrate changes to a branch (say Branch1) that is behind master by some commits, use the rebase command.

For instance, a second branch (Branch2) that was just integrated into master, making master ahead of Branch1. 

git checkout a_branch_that_is_behind_master_branch1
git rebase master

Conflicts

There could be conflicts. This is due to updates to master branch, for example. Git doesn't know which version to keep. The version are usually referred to as ours and theirs.

To resolve conflict, Go to Actions -> Resolve conflicts -> Resolve using theirs (if you are happy to keep the changes in outside branch)

Here theirs refers to the other branch (not master) as you are currently in master following git checkout master
