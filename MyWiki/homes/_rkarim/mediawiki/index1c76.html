Obtain scar shell

  • Use scar3d to obtain scar shell. Load LGE and then MRA. Export other shell as paraview, choose 1, 2, or 3. It doesn't matter as only the geometry is required.
  • Load the shell in Paraview and decimate it (use about 0.7 to 0.9 reduction in topology) - save the decimated shell
  • Run the voltage application. This is also saved up in ITK_TOOLS so should run as normal if called from the command prompt.


April 5, 2011

Alternate shell generation

  • As the LA surface from scar3d generates holes when it is decimated in paraview, the alternate way is to generate the shell from atrium3d. Atrium3d employs a range of mesh smoothing approaches (such as Laplacian) where shells even after decimation does not have many (hardly visible) holes.
  • Here are the steps to do this:

Prior to these steps ensure that the CJ_Seg_post and MRA_post have same image dimensions. If not, it is necessary to do a dummy transformation of the MRA_post to the CJ_seg_post's image space via: 

transformation Cj_seg_post.nii MRA_post.nii mra_post_in_cj_seg_post.nii -dofin dummy.dof -target mra_post

To get dummy.dof, run rview and obtain using 'save transformation'.

We note the steps to obtain shell from atrium3d. 

threshold CJ_seg_post.nii post_gad_seg.nii <thrshold value from rview examination> 
mymask MRA_post.nii post_gad_seg.nii post_gad_seg_mymask.nii 
atrium3d .

Next, load post_gad_seg_mymask, twice as CTRL+'1' and '2' and Export Surface as, option (2)
