Issues

Not finding Boost thread

  • This post seemed to help. Here in this post, the author says to hack into FindBoost.cmake (this file can be found in CGAL/cmake/modules folder) and add the following lines 
I use XP and cygwin and I fought for days with the issue of cmake not 
finding boost thread. 

It finally worked when I hacked the following into FindBoost.cmake, i.e. 
gave him the path manually. 

... 
SET(Boost_${UPPERCOMPONENT}_LIBRARY 
/usr/local/lib/libboost_thread-gcc34-mt-1_38.a) 

message("${Boost_${UPPERCOMPONENT}_LIBRARY}\n") 

message("${Boost_${UPPERCOMPONENT}_LIBRARY_RELEASE}\n") 

message("${Boost_${UPPERCOMPONENT}_LIBRARY_DEBUG}\n") 

_Boost_ADJUST_LIB_VARS(${UPPERCOMPONENT}) 

IF( Boost_USE_STATIC_LIBS ) 

SET(CMAKE_FIND_LIBRARY_SUFFIXES ${_boost_ORIG_CMAKE_FIND_LIBRARY_SUFFIXES}) 

ENDIF( Boost_USE_STATIC_LIBS ) 

ENDFOREACH(COMPONENT) 

# ------------------------------------------------------------------------ 

# End finding boost libraries 

# ------------------------------------------------------------------------

Note: the lines starting from _Boost_ADJUST_LIB_VARS(${UPPERCOMPONENT} are already present in the FindBoost.cmake file.

Linker error relating to gmp-vc80-mt.lib

  • Basically the library does not exist in the CGAL\auxiliary\gmp\lib folder. A forum thread I found addressing exactly the same issue. Solution is to rename the libgmp-10.lib to gmp-vc80-mt.lib. It's funny how that would fix things - but it worked!


Integrating CGAL with VTK + ITK projects

In CMake, three things for integration of CGAL:

  • Specify where the header files are located
  • Specify static libraries folders
  • Specify static library names

Steps First: In CMake, for INCLUDE_DIRECTORIES, these need to be added. Note this includes paths to Boost Pro, TAUCS solver and GMP. These are all found in the auxillary folder of CGAL installation (build folder): 

 INCLUDE_DIRECTORIES
(
 "C:\\PhD\\software\\CGAL_v3_8\\include"
 "C:\\PhD\\software\\boost"
 "C:\\PhD\\software\\CGAL_v3_8\\auxiliary\\taucs\\include"
 "C:\\PhD\\software\\CGAL_v3_8\\bin\\include"
 "C:\\PhD\\software\\CGAL_v3_8\\auxiliary\\gmp\\include"
)

And then specifying the directories the static libraries: 

 LINK_DIRECTORIES(
 "C:\\PhD\\software\\CGAL_v3_8\bin\\lib\\release"
 "C:\\PhD\\software\\boost\\lib"
 "C:\\PhD\\software\\CGAL_v3_8\\auxiliary\\taucs\\lib"
 "C:\\PhD\\software\\CGAL_v3_8\\auxiliary\\gmp\\lib"
)

And finally, the static libraries: 

TARGET_LINK_LIBRARIES( <your-other-static-libs> CGAL_Core-vc80-mt.lib CGAL_ImageIO-vc80-mt.lib CGAL-vc80-mt.lib libatlas.lib 
libcblas.lib libf77blas.lib liblapack.lib libmetis.lib libtaucs.lib libtstatlas.lib vcf2c.lib libboost_date_time-vc80-mt-1_42.lib 
libboost_date_time-vc80-mt-s-1_42.lib libboost_date_time-vc80-s-1_42.lib libboost_thread-vc80-mt-1_42.lib
 libboost_thread-vc80-mt-s-1_42.lib gmp-vc80-mt.lib libgmp-10.lib libmpfr-4.lib )
