From PatientCoordinate To VoxelCoordinate

From Rashed Karim Wiki

Jump to: navigation, search

Introduction

Osirix MPR xyz (which is in patient co-ordinate system) to image voxel space. Patient co-ordinate system is in XYZ mm and image co-ordinate system is in number of voxels in xyz from (0,0,0)

Osirix MPR views

  • The 'image size' in the MPR view, changes when you re-size the MPR GUI window. So it seems to be doing some resampling on the fly, and thus those those numbers cannot be relied upon.
  • The XYZ in mm in MPR seems to correspond to the XYZ in mm in the image view. So this, can be relied upon.
  • From the above point, it is technically possible to derive the voxel XYZ from MPR XYZ. A matrix will allows us to achieve this.

Matrix construction

The matrix is a 4x4 matrix constructed from the DICOM headers. The following DICOM headers are needed (from my CT dataset, should correspond or be similar to yours):

  • ImageOrientationPatient - three values (xyz)
  • ImagePositionPatient - three values (xyz)
  • PixelSpacing - two values (xy)
  • SpacingBetweenSlices - single value

From the above, it is a matter of translating, scaling and sometimes rotating the MPR XYZ point to give the voxel XYZ point. This 4x4 matrix takes care of all that.

To cut a long story short, the MATLAB script below does the transformation.

For the script to run, you need to read the above tags from the DICOM header and put the required values into the script. I have left comments in the script.

And then you simply run the script as: P = MPR_2_voxels([-10.54 114.67 -193]);

The above is for an MPR point (-10.54 114.67 -193)

MATLAB code 



%{
    Converts MPR xyz to Image xyz

    Please change the constants inside the function to suit your DICOM
    image 
    O_x             - DICOM tag ImageOrientationPatient, x direction (direction cosine for
    X-direction). Example: [1 0 0]
 
    O_y             - DICOM tag ImageOrientationPatient, y direction (direction cosine for
    y-direction). Example: [1 0 0]
    
    Off_xyz         - DICOM tag ImagePositionPatient. Must be from first slice in image. Example: -125 -26 -133]
    pixelspace_xy   - DICOM tag PixelSpacing. Example: [0.48 0.48] 
    spacing_z       - DICOM tag SpacingBetweenSlices. Example: 0.45 
    

    mpr_xyz         - The input MPR xyz point. Example: [-10.5 114 -193]
    
%}
function P = MPR_2_voxels(mpr_xyz)
    
    
    O_x = [1 0 0]; 
    O_y = [0 1 0]; 
    Off_xyz = [-125 -26 -133]; 
    pixelspace_xy = [0.48 0.48]; 
    spacing_z = 0.45; 
   
   O_z = cross(O_x, O_y); 
   T = [O_x(1) O_x(2) O_x(3) 0; O_y(1) O_y(2) O_y(3) 0;  O_z(1) O_z(2) O_z(3) 0; Off_xyz(1) Off_xyz(2) Off_xyz(3) 1]'; 
   
   P1 = T \ [mpr_xyz(1) mpr_xyz(2) mpr_xyz(3) 1]'; 
   
   P = P1 ./ [pixelspace_xy(1); pixelspace_xy(2); spacing_z; 1];
   

end
