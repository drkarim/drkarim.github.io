IRTK Registration tools

A probabilistic atlas created for the structure showing the transversal, coronal and sagittal planes

IRTK [1] has an excellent resource of tools for registration. This includes rigid, affine and non-rigid registration. There are also some nice tools such as surface registration, the one commonly used by the face-recognition community for registering surface mesh points on a face. I have been primarily working on IRTK for the recent weeks to make a probabilistic atlas of an anatomical structure. Here is some guidance on how to construct an atlas.

Atlas construction

Now here is how we can construct an atlas:

We have N segmentations and in each segmentation we have selected the landmark points that correspond in all N images. We also select one of the segmentations to be our target shape/image T:

  • First point affine registration to transform segmentation_i to T (1<=i<=N):
    • pareg T_lm.vtk segmentation_i_lm.vtk -dofout pareg.dof
  • And then apply the registration parameters obtained:
    • transformation segmentation_i.hdr segmentation_i_transformed.hdr -dofin pareg.dof

Now at this stage we have nicely aligned segmentation_i to target T, however, we can do better using surface (rigid or affine) registration:

Surface registration

  • Using the marching cubes algorithm, create a dense mesh of landmark points (pseudo-landmarks) on the target shape. We also do the same for the segmentation_i_transformed.hdr shape. We call these two surface meshes: T.vtk and segmentation_i_transformed.vtk
  • We invoke surface (affine or rigid) registration tools in IRTK:
    • srreg T.vtk and segmentation_i_transformed.vtk -dofout srreg.dof
  • Finally we transform segmentation_i_transformed using the surface registration parameters (obtained in the previous step):
    • transformation segmentation_i_transformed.hdr segmentation_i_final_warp.hdr -dofin srreg.dof -target T.hdr


Warping the Atlas

  • We assume that an atlas has been already constructed using the atlas program (in iRTK). Usually this is just an averaging of the grey-values, preceded by a Gaussian blurring.
  • Picking the landmark points on the atlas (saved as a .vtk file) - we call this atlas_lm.vtk
  • Picking the same landmark points on the unseen target - we call this unseen_target_lm.vtk
  • Using point affine registration:
    • pareg unseen_target_lm.vtk atlas_lm.vtk -dofout pareg.dof
  • pareg.dof has the transformation parameters
  • Now transform the atlas using pareg.dof:
    • transformation atlas.hdr transformed_atlas_to_target.hdr -dofin pareg.dof -target unseen_target.hdr
