Scenario

A claim came late for a client. Invoices had been generated already and:

(important pre-requisites)

  1. This client didnt have any invoices generated for that month.
  2. It is also important to note that the tutor also did not have any claims on that month.

If the above pre-requisites are not met, perhaps it is a case of where this claim needs to go to an existing invoice

Database tables

Student_invoice

  • insert a new row (using insert option phpmyadmin).
  • ID is auto-generated. student_id is easily obtainable.
  • date_generated is same date as the invoice date, which invoice month do you want to add it to?
  • total_amount - total of all the claims that are going to go into it. The total the client is charged at
  • is_emailed, is_paid and date_paid all default

Claims

For the claim that is going to be inserted into the invoice, make this change:

  • awaiting_approval - 0
  • approved_on - could put the same date as the date_generated of invoice, assuming C approved it on the day the invoice was generated
  • student_invoice_id - obtained from the insertion above.
  • claim_amount - the amount to be paid to tutor.

invoice_to_claim

This table are invoices for tutors and not students, and thus keeps tracks of tutor's side of invoices. The invoice_id is not meant to match with student_invoice table ids.

  • Insert a new entry. Manually insert invoice_id (this is not auto-generated). Find out what's the next invoice_id. Make sure that the invoice_id inserted is not a clash. It is not primary key and thus DB does not check for it.
  • Get claim id from above

invoice_to_month

  • Similar to invoice_to_claim table.
  • Insert date (same as student invoice generation date) and is_paid and date_paid to default
