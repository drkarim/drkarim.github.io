Scar3d on ventrcle

Adjustments

Note: that it is assumed that the myocardial segmentations are available and this is different from the atrium case, where only the endocardial segmentation is assumed by the Scar3D software to be available. Thus these are assumed to be available for the ventricle data:

  • Binary mask of the myocardium
  • DE-MRI data with high signal intensity depicting scar

Input adjustments

  • In scar3d.cxx, In graphcut_segmentation_assist function, option_i == 5, computeStatsForABBloodPool in graphCutClassifier.cc, the inner blood pool is necessary to compute the blood-pool intensity statistics.
  • Also if _WHAT_IS_HEALTHY_TISSUE == 1 then inside computeStatsForABBloodPool( ) there is a call to collectStatsForDilatedSeg( ), thus the dilated segmentation is necessary


Scar3d

  • Run prescar3d on _le.nii and _gad_seg.nii.
    • _le.nii is the late-enhancement scan readily available.
    • _gad_seg.nii
  • The _blood_pool.nii and _atrial_wall.nii are not usable and needs to be replaced by manual segmented blood-pool generated. Use mymask.exe.
  • Replace _blood_pool and _Atrial_Wall using mymask.exe
