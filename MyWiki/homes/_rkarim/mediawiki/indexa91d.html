CMake

Cmake installation required. At the time of this writing, I used 3.3 and 3.4

VTK

Compile and install VTK using the instructions provided on this page, configure using CMake, then first build using make and then make install'

Other dependencies

The following dependencies were easily installed using the magnificent Homebrew package.

  • FLTK
  • libpng
  • jpeg
  • Boost
  • GSL - (GNU Scientific library)

To check whether Homebrew can install do: 

brew info FLTK

and then: 

brew install fltk

Note, I installed 'Boost without Python, but here is a blog post for steps compiling Boost with Python

Preparing IRTK

  1. The source code can be downloaded from the GitHub page
  2. When CMake is first run, choose to specify your own compiler (ideally). Not sure if it makes a real difference as g++ and gcc are aliases for clang apparently on MacOS. This was to avoid using clang. The path to g++ and gcc for CXX and CC files respectively are /usr/bin/g++ and /usr/bin/gcc
  3. Configure CMake, due to Homebrew installation, Cmake finds Boost, FLTK, etc library automatically. See my post on Stackoverflow about Brew install
  4. There is an error in Cmake configuration file for IRTK. This error is to do with a line 183 in <IRTK_source>\config\IRTKInclude.cmake FindVTK. See my stack overflow query on this. See commented-out: 
IF (BUILD_WITH_VTK)
   # Add VTK
   #INCLUDE(${CMAKE_ROOT}/Modules/FindVTK.cmake)   COMMENT THIS LINE OUT 
   FIND_PACKAGE(VTK)    # INCLUDE THIS LINE
CMake Linker flag settings to find libpng binaries
  1. In MacOS Sierra at least, you need to add a -L/usr/local/lib flag to CMAKE_EXE_LINKER_FLAGS. Check the box Advanced, and these options will appear. Otherwise the linker fails when it starts to compile the binaries for the applications, at about 40% into the compilation process. The clang linker on MacOS Sierra cannot locate the libpng library binaries located in \usr\local\lib. I had to add it manually. See my Question on stackoverflow

Compiling IRTK

  1. Compile using make.
  2. There will be errors in the compilation:
    1. Change SetInput functions to SetInputData: if using VTK 6.2, then this will only be to do with the vtkPolyDatWriter class and the SetInputData function, where the SetInput function is now deprecated as of VTK 5 and above. The fix is to change each line where compilation failed to: 
writer->SetInputData(someObject)   // was before writer->SetInput( ... )

There will be multiple instances where the compilation will fail due to this, and you have to individually edit each file and change the function name. No need to run CMake again, just run Make and it will continue from where it left


  1. Compilation should complete and library is ready to go!
