Surface parameterization instructions

From Rashed Karim Wiki

Jump to: navigation, search

Step 1

Generate LA surface mesh with scar information

  • Run Scar3D software and export VTK
  • Lets call this file as: 
surface_mesh.vtk

Step 2

Convert the surface mesh to OFF format for CGAL

  • Run rk_magic with --toCGAL switch to convert VTK to off 
magic2 -i1 <vtkfile> -o <off_file> --toCGAL x

Step 3

Draw cut in surface mesh

  • Open MeshLab and open the mesh and draw the cuts. Cut PV and cut around the mitral valve
  • Save as OFF call it: 
 <surface_mesh>_cut_NN.off
  • Run ReMesh software to clean the mesh above, open mesh and say yes at prompt. Save as off file call it: 
 <surface_mesh>_cut_NN_Cleaned.off

Step 4

Compute planar mapping

  • Run code: 
rk_test1 (or unfold.exe) <surface_mesh_cut_NN_cleaned.off> <eps_output.eps> <surface_mesh.vtk>

If the parameterization fails and it complains that the Mesh is not valid, then re-mesh or re-cut. Sometimes this seems to work.

New method: If mesh fails to be read by unfolding software rk_test1 then clean the mesh using meshlab only. Load mesh into meshlab. Filters -> Clearning and repairing -> Remove duplicate faces + remove duplicate vertex + select self intersecting faces

Step 5

Visualize Planar map

  • Run code for visualizing the planar map
  • Command:

Note vtkOutput.vtk, is the palanar map and is generated from Step4. uv_xyz_mapping.txt and uv_xyz_pointID_max.txt are the planar mappings and also generated from Step 4. 

seeunfold3.exe <unfold-vtkOutput.vtk> <surface_mesh.vtk>

Step 5a

  • 'Optional: Smooth the map using smoothscalars.exe: 
smoothscalars vtkOutput.vtk temp.vtk 2
magic3 -i1 temp.vtk -i2 vtkOutput.vtk -o vtkOutputSmooth.vtk --transferscalar x -t1 1 
del temp.vtk

Output file specification

  • vtkOutput.vtk - The 2D flattened map
  • uv_xyz_pointID_max - The maximum point IDs in 2D and 3D surface. There are two numbers in this file, the first is 3D and the second one is 2D
  • uv_xyz_mapping - The mapping from 3D to 2D. An excerpt from the file is shown below. 5 7109 refers to 5 on unfold maps to 7109 on 3D. 
5	7109
6	7109
7	7187
8	7109
9	7116
10	7116
  • All of this can be understood if you read the function ReadUVMappingFile which reads reads and digests this file in QuickFlattener class. The code library is here.
