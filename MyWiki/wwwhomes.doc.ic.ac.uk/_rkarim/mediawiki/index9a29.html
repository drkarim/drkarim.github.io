Introduction

Some general ideas about primary and non-primary ablation points

Generating FTI

  • FTI maps are generated as normal using the old technique. See above or use command (for parimary map): 
scar3d -eAB ab.txt -eFTI fti.txt -et <x mm> -es LA_surface.vtk -ep x -eInter 1 -o fti_x_mm.vtk
  • Note LA_surface.vtk is mostly called carto.vtk
  • Scar3d loads, go to Endosense -> Compute FTI map

Generating scar maps

  • Scar maps are generated using either graph-cut or MIP or manual, BUT on the post shell.
  • They can both be generated quickly using the auto mode: 
scar3d -w . -i1 <gad_seg.gipl> -i2 <graphcut_or_le> -m 0 -s 1 -o shell.vtk --ben x

Registering Scar shell to FTI in pre shell

  • The post shell is registered to the pre shell on which FTI is visualized using landmark and ICP. See here for the steps
  • Use rkapp1 registration software and load target shell as FTI and scar as source. Register using Landmark followed by ICP and transfer scar data to pre shell (containing FTI data): 
tkapp1 -w .
