Scar3D software

  • Ensure that the LGE and 3dheart segmentations are in the same image space. LGE should in the same image space as 3dheart. So, basically having equal number of voxels.

Hand segmentations

  • Obtain the hand-drawn segmentations - apply a closing filter - about 3 iterations
  • Run each hand-drawn segmentation by scar3d. Load LGE and load hand-drawn as auto-seg (Ctrl + 2). Set normal blood pool manually to mean, var = -1, -1.
  • Save as VTK file (for later analysis) - choose option 3 (original image)

Pre-processing for graphcuts

  • Before running scar3d, the search space must be generated.

Graph-cut auto segmentation
I have used auto-generation of pre-processed graph-cut files for this.

  • run prepareforgraphcuts <lge_scan> <gad_seg>. At the time of this writing, prepareforgraphcuts was kept in test1.cxx. This generates all the atrial wall, spatial prior, innerbloodpool files.
  • Run scar3d . and select scar -> compute graph cuts using pre-computed (using automatic file loading)
  • output is written to segmentation_output.gipl or .nii depending on the filetype specified.
  • the output undergoes the closing operation to fill gaps, 4 iterations normally.
  • output is blurred with a filter of kernel size 1.0
  • As blurring converts the otherwise binary output to a greyscale, it is necessary to apply a threshold at this stage. Normally a small threshold of 3-8 works well. Check in rview over late-gad scan to see which threshold works best
  • the thresholded blurred and closed segmentation output is also stored as segmentation_output_closing_blurred.gipl in the folders.

Changes

  • August 2011
    • MIP is no longer based on interpolating sub-voxel intensities. The 1-pixel neighbourhood around the normal is traversed to find the maximum intensity voxel to be MIP-ed onto the shell.
    • The colour scale used to be calibrated based on the maximum intensity MIP-ed. This can be an issue if the maximum intensity is from a noisy pixel that happened to have a high intensity value. Now the colour scale is based on the mean of the top N (e.g. 98%) intensities.
    • The ability to limit or expand the extent of the reach of the normals. This is the 'search space' of the surface normals.
  • September 2011
    • Setting the image folder path. Works on my installation
    • On some installations it crashed when the folder was set. For these cases, system can still work without setting the image folder. It finds the path from the first image that is loaded and makes this as the working directory.

Improvements

  • The update of the scar model during the iterative segmentation process can be improved. Sometimes updates can cause to move away from the optimal model. For example I start seeing more sink nodes (scar voxels). Perhaps the update can be clever when it sees a divergence or surge in number of sink nodes, it retreats from moving in that direction.
  • Region acceptance and removal
