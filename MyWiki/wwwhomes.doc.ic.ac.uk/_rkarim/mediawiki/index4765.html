Steps

  • The invoice generation process remains the same, except:
  • * New due to quick invoicing in 2015: Remove any quick invoice claims that are duplicated in case by tutor. Quick invoice registered claims and tutor re-registered same claim.

Once invoices are generated, follow these steps (OSX instructions here)

  • In 'Manage invoices' select the new month, view invoices
  • Start shell prompt, login to server files, go to Admin -> invoices_html and remove all html files (if any)
  • Select to view each individual invoice in-turn. On selection the 'HTML write to file on disk' module is invoked.
  • The files are now written into admin -> invoices_html
  • Use FileZilla, to copy files to local machine
  • Put them in local dropbox folder admin -> invoices_html. NOTE: It is important to put them in the dropbox folder so that the relative URL for the logo in the invoice works
  • Go to desktop invoice_processing folder, and run the convert.sh shell script: 

#!/bin/bash
FILES=/Users/rashed/Dropbox/web_projects/desilvatutors/admin/invoices_html/*
for f in $FILES
do
        filename=`basename $f`
        only_filename="${filename%.*}"
        only_extension="${filename##*.}"

        wkhtmltopdf "$f" "$only_filename.pdf"

done
  • Upload PDF generated in the desktop folder back to admin -> invoices
  • Prepare to now email the invoices
  • New due to quick invoicing in 2015: Do not email clients receiving quick invoices unless new hours are present after quick invoice. To do this automatically:
  1. In Student_invoice class, in function exclude_emails($str) , in $exclusion add to the email list
  2. It is important to reset this list after invoice dispatch is completed, otherwise it prevents invoice dispatch to excluded clients in subsequent months.

GIT handling

Tips:

  1. It is best to change code remotely during invoicing, for example invoice dispatch email address.
  2. Once, invoicing is completed: 
On remote:
git add <file.php> 
git commit 
Add message such as 'usual changes to code during invoicing on <month> 

On local: 
git pull;

If the master branch is complaining that it cannot checkout to phantom branch due to SudentInvoice.php, then try this: 

git checkout -- StudentInvoice.php

and then: 

git checkout phantom

Problems with terms.pdf

  • It has been noted that switching to branch phantom, gets rid of terms.pdf, this can be a nuisance
  • I tried switching to a new branch phantom2 and did not have problems with terms.pdf disappearing

SQL query to get recently linked tutors 

SELECT S.name AS 'CLIENT', T.name AS 'TUTOR', st.link_added AS 'WHEN_LINKED' 
FROM student_tutor st, tutor T, student S 
WHERE st.tutor_id = T.ID AND st.student_id = S.ID 
AND st.link_added > '2018-01-01'
