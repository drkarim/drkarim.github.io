Git workflow production and development

Experience with setup

  1. Deleted the previous .git folder in wwwdev and then git init
  2. Empty repo, included folders with git add. Folders: api, images, login, pages_iresus, pages_thinkglucose, pages
  3. Go to local and perform a git clone: git clone ssh://user@development:/path/to/git/repo (this is path to wwwdev), all folders are fetched
  4. Change permission on remote to allow push: using this stackoverflow query-answer 
cd /path/to/repo.git
chgrp -R groupname .
chmod -R g+rwX .
find . -type d -exec chmod g+s '{}' +
  1. Now any changes on local are pushed to remote using: 
git pull; git push
  1. But there is a problem with the above, where git complains that the remote master branch is checked-out. So simply checkout into the phantom branch using: 
git checkout phantom

And then push the changes. This was recommended in this stack post

  • The reason to do the above is due to the git error that is explained in this stack overflow query-answer. Note that the answer which worked was from the combination of two solutions:
    • Answered by ken orb to this post, and the
    • Another stack question to the same problem, but the best answer recommended to checkout to another branch if remote access was available.

Development and Release

  • On local computer after code change, do the following: 
git add <unstaged files> 
git commit 
On remote: git checkout phantom 
git pull; git push
  • Once happy with the changes and ready for release, remember to switch to MASTER branch on Development by: 
git checkout master
  • And then to release to Production, do: 
cd www 
git pull

Issues with GIT

The gitignore on the production folder is not really working. See this post that helped somewhat with diverging branches. This command helped to merge the Prod and Dev repo: 

git merge origin/master
