Orientation of Images

The orientation describes the mapping from the voxel coordinate system (i,j,k) to the physical coordinate system (x,y,z). In the voxel coordinate system, i runs along columns of voxels, j runs along rows of voxels, and k runs along slices of voxels. It is assumed (by the NIFTI convention) that the axes of the physical coordinate system run as follows: x from (L)eft to (R)ight, y from (P)osterior to (A)nterior, z from (I)nferior to (S)uperior.

Check orientation of your image

  • Load image into ITK-Snap
  • From menu, select Tools -> Reorient image

Change orientation

There are three tools for changing your image's orientation to the standard convention:

  • ITK-Snap
  • Convert3D --orient switch command
  • dcm2niigui

Tool 1: Convert3D

In the Convert3D documentation the --orient CODE switch changes the orientation of images to the NIFTII convention above.

The command to do this: 

c3d image.nii -orient LPI -o output.nii

The --orient LPI switch command changes the image to the NIFTI convention with x going from Left to Right, y going from Posterior to Anterior and z from Inferior to Superior.


Tool 2: ITK-Snap

Follow steps below:

  • Load image
  • Go to Tools -> Reorient image -> New orientation as LPI
  • Save image

Tool 3: dcm2niiGUI

In dcm2niiGUI, follow these steps:

  • Install MRICron
    • The files for the software come as a zipped file. There is no self installer in the link above; simply download zip file from the link, unzip contents and place it on a folder in your computer
    • Remember to select 'win.zip' windows version from the drop-down on the download page
    • Click on 'Download Now' link above the drag-and-drop.
  • Once drag-and-drop the image and use the 'reorient to orthogonal' in drop-down which appears.
  • The software writes the output image automatically to the same folder as input image, with an 'o' prefixed to the filename.
