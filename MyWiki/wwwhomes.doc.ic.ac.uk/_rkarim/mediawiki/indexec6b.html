Fixing corrupted DB after bad invoicing

How I reverted back the corrupted DB

[Dated: Oct 13] There are two phases, first is to identify which records need to be updated and removed. The next phase to perform the remove and update operations.

Record identification

  1. Thankfully all claims that were supposed to be invoiced, were approved on the day the invoice was to be generated. So they all had the same timestamp and a single query retrieved these claims. However, even if you are able to determine the claims that were supposed to be invoiced this month, you can move on to the next step.
  2. Retrieve the above claims, especially the claim ids. call the list of ids as CID 
 QUERY: select id from claims where approved_on = '<some date>'
. Note: Export the result of the query to Excel and then extract the id column and transfer to a text-editor to remove the \r\n characters to commas. Basically generating the ids as a comma-separated list
  1. Move to the invoice_to_claim table and nwo retrieve the invoice_ids here that match the above claim ids. Remember this invoice_id and the student_invoice_id are different. Call this list of ids as IDC 
 QUERY: select DISTINCT invoice_id from invoice_to_claim where claim_id IN ( ... ) <-- the list from above
  1. Move to the invoice_to_month table and check that the id list from above does correspond to the invoices that are eerroneous (not an essential step).

Now move to the student_invoice table. Retrieve the student_invoices that contain claims from list CID. Find the CIDs

Record update and deletion

  1. Delete all entries from invoice_to_month 
delete from invoice_to_month where invoice_id IN (1779,1780,1781,1782,1783,1784,1785,1786,1787,1788,1789,1790,1791,1792,1793,1794,1795,1796,1797,1798,1799,1800,1801,1802,1803,1804,1805,1806,1807,1808,1809)
  1. Delete all entries from invoice_to_claim 
delete from invoice_to_claim where invoice_id IN (1779,1780,1781,1782,1783,1784,1785,1786,1787,1788,1789,1790,1791,1792,1793,1794,1795,1796,1797,1798,1799,1800,1801,1802,1803,1804,1805,1806,1807,1808,1809)
  1. Delete relevant entries from student_invoice 
delete from student_invoice where id >= 2916
