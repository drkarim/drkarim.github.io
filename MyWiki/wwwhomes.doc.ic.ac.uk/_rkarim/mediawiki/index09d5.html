The Visualisation Toolkit (VTK)

VTK solutions

From time-to-time, as a frequent user of VTK, I come across many different types of issues primarily with integrating and using the basic software. I mostly resolve these through the now-excellent VTK forum supported by the growing VTK user community of experts.

  • How to get adjacent cells of a polygon, my query here.
  • Adjacent neighbouring points (of a point) in polygonal mesh here
  • Checking if a point is in a polygon here.
  • Labelling iso-contours here, and also here
  • Adding points to an existing mesh here
  • Using an icon to indicate image (world) orientation here
  • Shortest paths in a mesh using vtkDijkstraGraphGeodesicPath. However, this is not available in earlier versions of VTK, so I implemented one using Djikstra's algorithm. See here.
  • Drawing an ellipse here
  • Iterating through vertices of all cells in a polydata here
  • Visualize point clouds (stored in vtkpolydata) using paraview. Need to use the vtkMaskPoints filter in VTK/Paraview. See more info here
  • Selecting regions within a polygonal mesh here
  • Rendering multiple translucent objects to renderer here
  • Paraview exports (issue arised after ICP registration failed on paraview exported surfaces): After you apply a transformation or extract a selection to polygonal data in Paraview, the output is an unstructured point set. It is easy to convert this back to a polygonal mesh by applying a filter to 'Extract surface'. With this filter applied, the surface again becomes a polygonal mesh which can now be exported and read as polydata.
