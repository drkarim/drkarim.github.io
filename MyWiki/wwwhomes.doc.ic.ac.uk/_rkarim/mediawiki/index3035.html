Oct-26-2010

Added a flag _DECIMATE_ATRIAL_SURFACE to scar3d.h and scar3d.cxx for giving the option to choose decimation of atrial surface polygons. Previously, decimation was applied by default resulting in fewer polygons that made the scar surfaces pixelated. Changes also made to show_surface_normals in scar3d.cxx. Updated first in HP and then updated on Mac. Both previous versions of HP and MAC source backed-up on WD passport.

Oct-26-2010

Changes made to atrium in my_testers_with_irtk project. These changes have only been made to Mac laptop and not yet updated on HP. There were also changes made to tester1 and tester2. These are mainly changes to code to make it compile under Win7. For example, return types not specified.


Oct-29-2010

ONLY MAC OS Compiling my Atrium4D library under Mac

  • In base.h, had to change the REAL pi() declaration to REAL pi_b() as it name clashes with some pi declaration in Mac's cocoa library. Also need to make the same change in basis_r.cpp
  • In tree.hh, had to change the std::max declaration to just max(..), in line 1692.
  • myMath declaration of void sleep(..) clashes with Mac's internal library function int sleep. Changed it to sleep_t.
  • Changing all irtkBSplineImageFunction<irtkRealImage>& to just irtkBSplineImageFunction& bsp. Apparently, irtkBSplineImageFunction is not a template class. Checked class definition, and it appears to be so. So made the changes in vesselfinder.cc and vesselfinder.h
  • changed vtkPointerPicker* picker declaration to picker1 as name clashes with internal Mac function/class.
  • Removed template classes from irtkNearestNegborInterpolation in atrium3d.h, same as rtkBSplineImageFunction<irtkRealImage> complaining that it is not a template class
  • What surprise, apparently no itoa function in gcc/g++ compiler, so had to include an implementation of it found here in myMath.cc
  • G++ doesn't like casting void* to int*, so had to case void* to intptr_t instead. In many lines for atrium3d.cxx
  • Changed a declaration of double INFINITY to double INF. Not sure, perhaps name clash causing problem.


Nov 4-2010

Made changes to the windows version of code on mac laptop. Now scar3d accepts a second command-line param apart frm specifying working direction (first param). The second param is the threshold to be used in the marching cubes algorithm for generating atrium shells. For example, in binary images (of LA segmentation), Use: scar3d . 1. NOTE: No changes made to HP laptop version.

Nov-8-2010

Updated Win HP PC with Mac updated code. Compiled fine. All the Win PC code is backed up on WD passport (HP folder and backup) and also backup-ed on HP pc backup folder under PhD.

Nov-10-2010

Added a new EndosenseToolbox class to the source tree on Win laptop. No changes yet on Mac.

Nov-13-2010

Version 1.0 of EndosenseToolbox works on Mac (WIN bootcamp) - only has the very basic file output capabilites.

Nov-30-2010

Made major changes to graphcut classifier library. All the pre-computation can now be done off-line and pre-computed images such as atrial wall, spatial priors and inner blood pool images can be loaded without computing them in the software. The application to pre-compute these images is found here

Dec05-2010

Changes to name: VRMLFileReaderWriter to VRMLandVTKFileReaderWriter. Also included many VTK functions in this class which only previously had VRML functions.

April 2011

Added a functionality within tools to load multiple shells and analyze multiple shells using ROI. Analysis is now limited to reporting the mean and total cell (i.e. polygon) scalars within a Region of interest selected by a bounding box. Inclusion of functions 

*Load_N_Shells(Fl_Widget*, void* user_data) 
*analyse_N_shells(Fl_Widget*, void* user_data)

and in math.cc -> 

* analysePolyData(vector<vtkPolyData* > polys, vtkPlanes* ROI)
* displayPolyData(vtkPolyData* poly, vtkFlRenderWindowInteractor *&flrwi, bool freshRender, vtkActor* actor)

To use this tool, first tools -> load multiple shells -> enter number of shells to load. Next, press 'x' to select a ROI. Then, Tools -> analyze multiple shells. The stats are reported in the command line prompt.
