Introduction

Quaternions represent 3D rotations. Although, 3D rotations are complex transformations, but they can be simply represented by a rotation angle and a vector which is the axis of rotation. These two pieces of information are encapsulated in a special 4-vector known as a Quaternion. Quaternions are represented as complex numbers. This allows them to take advantage of complex algebra which is simple to use and learn. Two quaternions can be added using simple complex algebra and made to represent the much more complex 3D rotations in succession represented by the two quaternions.

IRTK and Quaternion

Here is a piece of code that demonstrates the conversion of a Quaternion to a 3D matrix using VNL and IRTK libraries:

Note: The VNL library converts the input quaternion to unit quaternion in its constructor 

Usage: quaternion.exe -t T -u U -v V -w W       // Q = T + Ui + Vj + Wk

Code 


#include <irtkImage.h>
#include <irtkQuaternion.h>
#include <irtkMatrix.h>

#include <vnl/vnl_matrix.h>
#include <vnl/vnl_vector.h>
#include <vnl/vnl_quaternion.h>
#include <vnl/vnl_transpose.h>

#include <stdio.h>    // to get "printf" function
#include <stdlib.h>   // to get "free" function



void GetQuaternionMatrixVNL(double t, double u, double v, double w)
{
	vnl_matrix_fixed<double, 4,4> m; 
	vnl_quaternion<double> *q1 = new vnl_quaternion<double>(u,v,w,t); 

	m = q1->rotation_matrix_transpose_4(); 
	m = vnl_transpose(m); 
	for (int i=0;i<m.rows();i++)
	{
		cout << "\t";
		for (int j=0;j<m.cols();j++)
		{
			cout << m.get(i,j)  << "\t"; 
		}
		cout << endl; 
	}
}

void GetQuaternionMatrixITK(double t, double u, double v, double w) 
{
	irtkQuaternion *q1 = new irtkQuaternion(t,u,v,w); 
	
	irtkMatrix m(3,3); 
	m = q1->ComputeRotationMatrix(); 

	for (int i=0;i<m.Rows();i++)
	{
		cout << "\t";
		for (int j=0;j<m.Cols();j++)
		{
			cout << m.Get(i,j) << "\t"; 
		}
		cout << endl; 
	}
}
