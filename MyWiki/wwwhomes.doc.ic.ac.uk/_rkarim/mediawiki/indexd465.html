Creating an installer

  • To the project tree (right at the top), right click:
    • Add > Add new project > Other project types > (select) Setup wizard and give it a name
  • A new setup wizard project is created. Now click on the setup wizard project > right click > view > Five different views are presented
    • Of the 5 views, the most important are 'File system' and 'User interface'
    • In 'File system' > under Application folder > right click > Add > Project Output > select which project's output you want to be packaged with this installer
  • You can customize the welcome message in the splash screens (i.e. when installer loads). Select 'user interface' view by right clicking on the setup wizard project:
    • Click on each of the different stages of the installer prompts. For example, to change the welcome message, right-click on Welcome > Properties > Change text in welcome properties.

Changing the name of the executable

Sometimes the project is named XYZ and you wish to name your executable as something else. Follow these steps to change the assembly name:

  • To change the exe file name output after build: Project Properties > Linker > General > Output File.
  • To change the product name: edit the version information in your resource file (.rc file).

Add shortcut to target user's desktop

  • Right click on setup project (in tree) -> view -> File system
  • Click on user's desktop folder on right
  • On the empty left side of the screen, right-click and then 'Create new shortcut' -> Application folder -> Primary output from project

Add other files e.g. icon, banner, missing DLLs to target PC

  • You wish to add files such as icons and other banners (used in the installation welcome screen). This is done by adding files to the application folder (under file system accessed using instructions for creating desktop shortcut above)
  • Adding missing dependencies: MSVC will frequently not find all the necessary dependencies. These can be found by trial and error:
    • Installing the software on the build pc (Program where you wrote all your code).
    • Browsing to the installed folder using command line
    • Changing path to just path=c: (removing all paths to dlls installed in build PC, i.e. creating the same scenario as target PC)
    • Executing the .exe and checking what error message pops out identifying the missing DLLs in-turn and copying them to the installation folder. This way all missing DLLs can be found
    • Include all these un-identified DLL dependencies to the application folder of the target PC.
