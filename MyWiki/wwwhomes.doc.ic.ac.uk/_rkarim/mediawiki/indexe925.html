Boost

Note the compiler assumed here is the Microsoft Visual studio C++ 2005 (ver. 8.0)

  • Download pre-compiled windows binaries from the Boost Pro website here
  • Once installed, you need to find where the headers, .libs and .dlls are for integration into your project:
    • For static and dynamic link libraries (i.e. LIB and DLL files): There should be a sub-folder under the installation called lib which has all the libs and dll files.
    • For Header files: This is all under the sub-folder called boost
  • If you have a CMake file in root of your project, you can need to include the header directory as below. Otherwise you can go to your project settings (Project->(your project name) properties->C\C++ (additional include directories (set include folder). and then Linker (set all .lib files below): 
INCLUDE_DIRECTORIES(
"C:\\PhD\\software\\boost\\binary"
)
  • And, then tell the linker where the find the linked libraries: 
TARGET_LINK_LIBRARIES( <your other linked libraries> libboost_bzip2-vc80-mt-1_47.lib 
libboost_chrono-vc80-mt-1_47.lib libboost_date_time-vc80-mt-1_47.lib libboost_filesystem-vc80-mt-1_47.lib libboost_graph-vc80-mt-1_47.lib 
libboost_iostreams-vc80-mt-1_47.lib libboost_math_c99f-vc80-mt-1_47.lib libboost_math_c99l-vc80-mt-1_47.lib libboost_math_c99-vc80-mt-1_47.lib libboost_math_tr1f-vc80-mt-1_47.lib libboost_math_tr1l-vc80-mt-1_47.lib libboost_math_tr1-vc80-mt-1_47.lib 
libboost_prg_exec_monitor-vc80-mt-1_47.lib libboost_program_options-vc80-mt-1_47.lib libboost_python-vc80-mt-1_47.lib 
libboost_random-vc80-mt-1_47.lib libboost_regex-vc80-mt-1_47.lib libboost_serialization-vc80-mt-1_47.lib libboost_signals-vc80-mt-1_47.lib 
libboost_system-vc80-mt-1_47.lib libboost_thread-vc80-mt-1_47.lib libboost_unit_test_framework-vc80-mt-1_47.lib 
libboost_wave-vc80-mt-1_47.lib libboost_wserialization-vc80-mt-1_47.lib libboost_zlib-vc80-mt-1_47.lib  )
  • IMPORTANT: Note, the that the libraries shipped in the download archive (i.e. found in the lib folder), all start with boost prefix. However, the MSVC linker searches for libraries that begin with libboost. If this happens, you need to rename all the .libs to include the prefix lib. Thus for example: 
boost_math_c99-vc80-mt-1_47.lib

becomes: 

libboost_math_c99-vc80-mt-1_47.lib
  • Finally, to finish off, you must include the 'lib folder in your system path, so that the binaries when run can find the '.dll' files.

wxWidgets

  • Download the source code from here. Note you need the MSW version and which stands (not surprisingly) for (MicroSoft Windows)
  • Under the build\msw folder there should be a visual studio project file called wx.dsw. Open this in Visual Studio. As this was configured in Visual C++ (as of writing Feb 2012), MSVC will ask if the project can be converted to the MSVC 2005 versions. Hit 'yes' at the prompts and then at some point 'yes to all'
  • Project tree opens up MSVC. Build and then Build Solution.
  • The only thing to do now is to integrate to your project. In CMake: 
INCLUDE_DIRECTORIES(
 "C:\\PhD\software\\wxWidgets_2_8\\include"
 "C:\\PhD\\software\\wxWidgets_2_8\\include\\msvc"
)

LINK_DIRECTORIES(
"C:\\PhD\\software\\wxWidgets_2_8\\lib\\vc_lib"
)
  • Try out a Hello world application as given here. Note: If you are running windows in console mode with the above hello world application, you must include a main function: 
int main(int argc, char* argv[])
{
	return WinMain(::GetModuleHandle(NULL), NULL, ::GetCommandLine(), SW_SHOWNORMAL);
}
