VTK and FLTK

  • VTK installed without any problems on Windows 7 running Visual studio 2005.
  • FLTK required some tweaking, especially if you want to compile with ZLIB and PNG libraries. If you have downloaded the source code of the libraries and compiled them, then you must set the USE_SYSTEM_ZLIB flag to off, telling CMake not to use system zlib libraries, and rather the libraries from your downloaded source. Correction: Better not to use system zlib,png and jpeg as FLTK source tree includes this libraries, also works well with IRTK compilation if done this way. However, in IRTK compilation I still used system PNG, ZLIB
  • It is important to use ZLIB and PNG libraries for compiling IRTK.

IRTK - Daniel Rueckert's ITK

Compiled without much problems after installing FLTK properly (using zlib and png libraries). Also important that CMake finds Glut libraries. One part of the code called sysmnreg did not compile, but the rest had no problems. All applications seem to run well such as rview, transformation, areg, rreg, nreg, etc.

The main libraries that have to be pre-installed before compiling IRTK are:

  • VTK - compiles easily (does not require dependency libraries)
  • FLTK - compiles relatively easily (again does not require dependencies)
  • GLUT (Best if the binaries are pre-compiled and copied from a machine, i.e. the .h and .lib)
  • libpng (can be downloaded and compiled using cmake from here
  • zlib (can be downloaded and compiled using cmake from here
  • OpenCV (not necessary, but if you do choose to use openCV libraries, can be compiled easily from downloadable source from this link

And then finally, it is matter of pointing to the correct include (where the .h files are) and the compiled libraries (the .lib files assuming Win34/64 OS) in the CMake settings. See CMake settings on the screen-shot.

My code

Code required some changes, notably:

  • Cannot declare functions without return types, must be declared as void. VC 2003 only threw a warning, but VC 2005 throws compilation errors.
  • in fegien.cpp, code 
     Register i
    must be changed to 
     Register int i
  • the pow function in cmath.h was used previously as 
     pow(i,j)
    but now must be changed to cast params to double: 
     pow ((double)i, (double)j)

Cmake settings on Mac Win PC

Cmake file with open cv, Daniel ITK, VTK and FLTK 

PROJECT(IRTK_ATRIUM)
FIND_PACKAGE(FLTK)
  IF(FLTK_FOUND)
    INCLUDE_DIRECTORIES(${FLTK_INCLUDE_DIR})
  ENDIF(FLTK_FOUND)

  FIND_PACKAGE(VTK)
  IF (VTK_FOUND)
      INCLUDE (${VTK_USE_FILE})
  ENDIF (VTK_FOUND)  
  
  FIND_PACKAGE(ITK)
  IF (ITK_FOUND)
      INCLUDE (${ITK_USE_FILE})
  ENDIF (ITK_FOUND)  
  
FIND_PACKAGE( OpenCV REQUIRED)
IF (OpenCV_FOUND)
	INCLUDE_DIRECTORIES(${OpenCV_INCLUDE_DIR_CV})
ENDIF (OpenCV_FOUND)
