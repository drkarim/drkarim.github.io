Unfold dense mesh and fusion issues

Jump to: navigation, search

Problems encountered during unfolding

The following problems have been encountered during unfolding:

  • Dense mesh: When the patients-specific mesh has a large number of vertices (>10k)
  • Mesh fusion: The patient-specific mesh cannot be registered to the mean mesh, due to point-picking in my mesh fusion software.

Dense meshes

When the patient-specific mesh contains a large number of vertices, the number of vertices in the truncated mean mesh should be increased by using the VTK and Paraview Subdivision filter. These steps should be followed:

  1. Subdivide the mean mesh (.vtk): the mean mesh has 5k vertices and the next subdivision processes generate about 30k, 80k and so on.
  2. Convert to CGAL's Object file format (.off) using magic2 program.
  3. Register the patient-specific mesh to the mean mesh and transfer the scalar data to the mean mesh using the cartofuse program

Dense mesh long computation

In the unfold mapping code, these lines do not effectively construct the unfold mesh cells (i.e. polygons) from the CGAL output, as they run in polynomial time: 

  Halfedge_facet_circulator j = i->facet_begin();
        // Facets in polyhedral surfaces are at least triangles.
        CGAL_assertion( CGAL::circulator_size(j) == 3);
        //std::cout << CGAL::circulator_size(j) << ' ';
        do {
			pointFound = false; 
		
			for (int k=0;k<point_list.size() && !pointFound;k++)		// determine point's id 
			{
				if (mesh_adaptor.info(j)->uv() == point_list[k]) { 
					pointIDs[pointIDIndex++] = k;			// point found, retrieve ID and store 
					pointFound = true; 
					numPointsFound++; 
				}
			}
        } while ( ++j != i->facet_begin());
		
		CGAL_assertion( numPointsFound == 3);
		cells->InsertNextCell(3, pointIDs); 
		//scalars->InsertTuple1(faceCounter++, 1);		// all scalars default to 1 at the moment - no scalar information 
		
		faceCounter++; 
    }

One solution is to not compute the cells (polygons) in the unfold VTK output. Instead, output a point cloud of the unfold, represented with a VTK Unstructed Grid (.VTU) structure.

I have written the code for generating unfolds that are point clouds.

Fusion issues

The patient-specific mesh cannot be registered to the mean mesh due to point-picking issues with the mesh registration software.

To solve this issue, the following steps were followed:

  1. Register mean mesh to patient-specific mesh (i.e. the reverse), save 'Transformed source shell' option.
  2. Copy scalars from patient-specific mesh to mean mesh (transformed), using closest distance, use TransferScalar program as: 
TransferScalar -i1 patient-specific.vtk -i2 mean_in_patient_specific.vtk -o mean_in_patient_specific_with_data.vtk
  1. Copy scalars from mean mesh (transformed) to mean mesh using direct vertex copy, i.e. based on matching corresponding point IDs. 
TransferScalar -i1 mean_in_patient_specific_with_data.vtk -i2 mean_mesh_30k_nodes.vtk -o mean_mesh_30k_nodes_with_data --directcopy x
