CMake

CMake setting was fairly straight-forward. I failed to install FLANN due to Homebrew issues not able to write to /tmp folder.

CMake Settings for MIRTK

Compiling

Compiling was straightforward. Make a note of where the targets are installed. I installed them in '/opt/mirtk', but make sure the folder exists.

Use native Xcode compilers: 

make 

make install  (need sudo access to write to /opt/mirtk)

Installation

The targets were all installed in /opt/mirtk/lib/tools

Writing applications using MIRTK

Moving from IRTK to MIRTK is fairly straightforward if the new API is understood. i have already ported a few applications from IRTK to MIRTK and you can see the differences here:

  1. Simple segmentation of atrial wall: Code in IRTK and the code ported into MIRTK
  2. Dice Similarity Index per slice: Code in IRTK and the code ported into MIRTK
  3. Watershed segmentation: Code in IRTK and the code ported into MIRTK

For example: irtkGreyImage is now GreyImage.
