Visualizing ablation tag points

Endocardial views of the ostium in reconstructed left atrium segmentations.
  • Load LA segmentation shell, using Ctrl+'1' (LA_seg.gipl) and Ctrl+'2' (LA_seg.gipl). Enter -1, -1 for normal myocardium.
  • In endosense menu, select 'Load ablation point and FTI'
  • When asked to load ablation tag files, load abtTag.txt. Each line in this text file specifies the ablation tag point (x,y,z) in the world co-ordinate (roughly). So these points lie roughly on the surface of the LA.
  • When asked to load the FTI file, load the FTI file that is created manually listing all FTI force values in sequence. So it is very important the number of lines in this file matches the ablation tag file. Also the first line of the FTI file must specify the number of points, such as Number: 150
  • The program then displays the 3D shell and displays the ablation tag points (i.e. the points at which the force was recorded and ablation made)

FTI distribution on Mesh

  • STEP 1: Generate LA mesh:

Prior to running the FTI mapping tool, it may be necessary to generate the LA VTK surface using atrium3d see instructions here.

This is important as some VTK formats (especially those generated using ITK-Snap) is known not to work. To do this step, open the LA segmentation in atrium3d with CTRL+'1' and then again using CTRL+'2'. Then File->Export surface mesh as VTK

  • STEP 2: Distribute FTI on LA mesh:

The automatic tool that runs from the command-line for distributing FTI using for example a Gaussian-decay primary ablation force map: 

scar3d -w . -eAB ablTags.txt -eFTI fti.txt -es la_surf.vtk -et 5 -ep x -eInter 3 -o gangmap.vtk

The comamnd line arguments are explained here:

  • -eAB - the ablation tag points text file listing the x,y,z world co-ordinates.
  • -eFTI - the corresponding fti contact-force for each of the points in ablation tag
  • -es - the LA surface geometry in vtk format
  • -et - threshold for defining primary ablation points, for e.g. 10 means all points within 10mm of the ablation point are discarded making it the primary ablation point, starting from first point in the ablation point list. Note: the force is then also distributed within this specified radius (based on the method you request through -eInter).
  • -ep x - Requesting that only primary ablation points be used to generate force map
  • -eInter [1,2,3] - The interpolation type: 1) constant, 2) linear (force decays linearly), 3) Gaussian exponential decay
  • -o - the output of the interpolated contact-force mapped to the LA surface geometry

Note after specifying these params, the scar3d GUI loads, and you select from top menu Endosense -> Compute FTI map.

What to do if crashes:

Note if the interpolating software crashes, as this computes shortest paths using djikstra (without utilizing vtk djikstra classes), there is a limit on the number of vertices it can process. Decimate the left atrial mesh to reduce the number of triangles/vertices. This was done in case2 of endosense.
