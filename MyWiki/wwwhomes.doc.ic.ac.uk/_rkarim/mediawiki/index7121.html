Creating phantoms for scar

Note: If you are looking for the ventricle phantom scar program see here

Step 1

The first step to creating a phantom, is creating a template. The program can be found here. A template is an image which indicates what regions in the image are blood-pool, myocardium and background. Here is an example:

You can create this phantom by providing the createPhantom program two things:

  • LE image
  • Binary segmentation of the LA from the MRA GAD or SSFP scan.

Note that the segmentation of the LA must be registered to the LE image. The command to generate the template is then as follows: 

createPhantom 1 <le-image-nifti-or-gipl> <gad-binary-segmentation-niftii or gipl> <output-phantom-template-file>

Step 2

Now we will draw the scar in the template image. Open the template in ITK-Snap software (i.e. the template generated in step 1). Draw regions of scar using the paintbrush tool. See image below

Figure: Follow steps 1, 2 and 3 on ITK-Snap to draw the scar on the template and then save as a segmentation. .

NOTE: We will call the file saved in step 2 in the remaining of the document as phan_scar.gipl

Step 3

If you have reached step 3, you should now have the following files:

  • LE scan (le.gipl)
  • GAD MRA left atrium binary segmentation (e.g. gad.gipl)
  • Phantom template (e.g. phan_template.gipl)
  • Scars drawn on the phantom template (e.g. phan_scar.gipl)

Now all you need to specify is the mean of blood pool, atrial wall, scar and background. For example, blood pool could be 100, atrial wall could 120 and scar could be 200 (note these are grey-scale values). So letâ€™s call these values as bp_m, aw_m , sc_m, bck_m. The final command to generate the phantom is: 

createPhantom 2 phan_template.gipl phan_scar.gipl phantom_output_filename.gipl bp_m aw_m sc_m bck_m sc_stdev

Typical values here: 

createDigitalPhantom 2 phantom_template.gipl phan_scar.gipl phantom.gipl 140 117 280 30

Note sc_stdev is the standard deviation for scar. phantom_output_filename.gipl is the phantom output file.

Step 4

Apply Gaussian blur to phantom.gipl in all directions, and in step 5 we do anisotropic blurring 

blur phantom.gipl phantom_blur.gipl 1.0

Step 5

Peform anisotropic blurring in the z-direction to simulate the consequence of having overlapping slices in the real MRI acquisition. Overlapping slices meaning the slice thickness is more than the slice spacing. This results in partial volume in the z-direction. Here is the code to blur the phantom in the z-direction only 

scaleSpace -o phantom_aniso.gipl -sc 2.0 -Gz phantom_blur.gipl

You can read more about this in another wiki entry

Step 6

Apply Gaussian noise with your favourite noise injection program to the phantom. The program to do this is: 

addNoise -i phantom_blur.gipl -o phantom_final.gipl -m 0 -s 1 -a 10

The code for addNoise can be found here.
