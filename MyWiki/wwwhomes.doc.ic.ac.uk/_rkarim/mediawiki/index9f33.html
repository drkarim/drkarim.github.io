Level set curvature methods for atrial body

See also an old page which lists the same content

The levelsetcuravtureThresh application can generate the atrial body. Some empirically good parameters known to work are given below: 

seed = 200, 238, 33.0 
initial distance = 5.0 (not sure if this is right)
low thresh = 2
high thresh = 1000 
curv const = 100.0

If you have the left atrium segmentation, let's call it _gad_seg.nii, the EDT (Euclidean distance transform) image can be computed using the tester2 program. Once this is computed, invoke the levelset program: 

levelsetcurvaturethresh _edt.nii _atrial_body.nii (x y z) 5.0 2 1000 100

Here (x y z) denotes the co-ordinates of the centre of the atrium as determined using ITK's imageviewer program. The 5.0 is the initial distance parameter. 100 is the curvature scaling. The rest are the lower and upper thresholds.

Vesselness function

For generating the vesselness response image, the 3dheart/gad or mra scan must be masked by the LA segmentation. This normally can be done using the mymask program 

mymask 3dheart.nii 3dheart_la_seg.nii 3dheart_masked.nii

Using this masked image, which contains the actual grey-level intensities from the MRI, the vesselness response can be computed: 

mutliscalevesselnessexample3d 3dheart_masked.nii vesselness_response.nii scales.nii 0.5 5.0 5

Here we specify the range of scales that need be explored for the vesselness response. Note that the maximum response across the entire scale range is recorded in the vesselness response output. So, in the above example we explore scales ranging from 0.5 to 5.0 with 5 steps between this range.


Compute the Ridgeness (Hui's ridgeness)

In Atrium3D software, load ctrl+'1' the 3dheart_seg or gad_seg file first, followed by ctrl+'2' to load the masked segmentation of the atrium (see above). We have called this 3dheart_masked.nii above. This should display the left atrial surface shell. Now load the EDT of the 3dheart using ctrl + '3'. This is essential for computation of ridgeness. Now, select the Vessel analyze -> Compute HUI Ridgeness for entire image to file. System then prompts to load the necessary vesselness and scale files. Scale file is requested first and then the atrial body found in step 1 using level-set functions.

NOTE: Must name the ridgness output file to be saved to disc as mri_ridgeness.nii (this will be used in later steps in auto-pilot mode)

Ridge tracking

This step tracks vessels by maximizing the ridegness function computed in the above step. For this rename all files accordingly for running the atrium3d is auto-pilot mode: 

edt -> mri_auto_seg_edt.nii 
gad_seg / 3dheart_seg -> mri_auto_seg.nii 
vsslness -> mri_vssl.nii 
atrial body -> mri_at_body_edt.nii

To run auto-pilot, Tools -> Start auto ridge tracking

The following prompts will appear:

  • System prompts for Upper bound on distance from atrial body (careful when entering this value, a too high value may detect the appendage to be a drainage). A value of 1000 maybe to high.
  • System prompts for show seeds, enter 'n'
  • System also prompts for lower bound, enter 10 or something

Finally, it is very important that the result of the tracking is saved by going to Vessel analyze -> Write Ridge to text file.

NOTE: The file here must be called firstRidge.txt, or the next step crashes.

To compose the vessel drainages

NOTE: For this step to be succesful, the vessel tracking file output from the above step must be called firstRidge.txt, or this step crashes.

Run atrium3D, Tools -> Left atrial vessel compose.

There are no manual steps involved in this step.
