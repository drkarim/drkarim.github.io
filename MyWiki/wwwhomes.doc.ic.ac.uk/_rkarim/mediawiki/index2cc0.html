Adding a new page

Assuming we are adding a new content_type page, say calendar


Calendar.php

  • The left menu, add a new list element and add the active class 
 <li class="active"><a href="....../calendar.php">Calendar</a></li>
  • Add div of class form-group elements inside NewEntryForm. Repeat for each new entry form field entry 
<div class="form-group">
 <!-- Title input-->
 <label class="col-md-1 control-label" for="title"><span class="pull-left">Message</span></label>
 <div class="col-md-11">
 <input id="inputMessage" name="inputMessage" type="text" placeholder="Enter message" class="form-control">
 </div>
</div>
  • Do the above for EditEntryForm form
  • Change the data table HTML id name: 
<table id="announcements_entries" class="table table-striped table-bordered" cellspacing="0" width="100%">

And accordingly, change the JQUERY onclick' event in document.read(): 

 $('#announcements_entries tbody').on( 'click', 'tr', function () {
  • IMPORTANT: Change content_type_ids everywhere to calendar, so for example: 
// php 
$table_data_as_jscript_obj_content_final = Datatable::GetContentFromServer($space_id, 'calendar'); 

// javacript
var content_type_id = 'calendar';

Calendar.js

  • Copy paste all functions from other content_type js

Function AJAX_new_entry

  • If there are any rich text html fields, these have to be formatted here, such as this line: 
var inputDescription = $('textarea[name="inputDescription"]').html($('#inputDescription').code());

Add one line per html field

Function AJAX_edit_entry

  • Add or change line for any html rich text field: 
 var inputEditDescription = $('textarea[name="inputEditDescription"]').html($('#inputEditDescription').code());
  • Change handler in ajax call: 
 request = $.ajax({
          url: "/someurl/announcement_handler.php",
          dataType: "json",
          type: "post",
          data: serializedData
      });

Function initTable

  • Change the column names to match that of what is received from Datatable.php. Column names must match what is in contentful 
{ data: null, defaultContent: '', orderable: false, "visible": false },
        {data: 'message'}, 
        {data: 'description'}, 
        {data: 'description_html', "visible": false},
        {data: 'originator'},

Function ResetFormFields

  • Change all field resets accordingly



Calendar_handler.php

  • Copy-paste from existing code xxx_handler.php
  • Change function names NewCalendar and EditCalendar.
  • Change content_type_ids to calendar in both functions

Function NewCalendar()

  • Change the post field index names, for example: 
$title = $post['inputTitle'];
$date = $post['inputDate'];

Function EditCalendar()

  • The changes are the same as the changes to NewCalendar()


Datatable.php

Functionality: This php file is responsible for fetching table row-data from contentful and converting to JSON to serve the AJAX request of calendar.php

Function GetContentFromServer

  • If there is an html field in the content_type, adding two if statements below:
    • Add an if statement to shorten html, note in this example below information field-name is from what contentful calls it. See its content-model. 
if ($content_type == "directory"  && strtolower($field_name) == 'information')     // in directory entries 
          {
            $field_value = Toolbox::shortenTextAndStripTags($field_value, 50);      // strip html tags and shorten 
            $field_value = clean($field_value);
          }
    • Add a second if statement for adding the raw html field to feed 
if ($content_type == "calendar" && strtolower($field_name) == 'information')
