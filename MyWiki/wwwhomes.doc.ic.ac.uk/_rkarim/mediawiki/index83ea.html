Graph-cuts segmentation

These are the steps that were required to process the miccai images

  • Create patient folder
  • Copy:
    • pXX_De.nii - DE-MRI image
    • pXX_myo.nii - myocardium binay segmentation mask image
    • pXX_myo_plus_Bp.nii - myocardium as 1 and blood-pool as 2. It is OK for bloodpool to overlap with myocardium. Infact even excessive overlap is fine as it will be corrected in the following step
  • Use the program here to fix the overlap of blood-pool with myocardium, thus: 
 FixCarelessBP.exe pXX_myo.nii pXX_myo_plus_bp.nii pXX_bp.nii
  • The above program generates a file called pXX_bp.nii.
  • Create a new folder called gc. Copy pXX_de.nii, pXX_myo.nii and pXX_bp.nii inside gc
  • Rename the files inside gc as follows:
    • pXX_de.nii to _le.nii (you are renaming this so that graph-cuts software knows what the DE-MRI scan is called, saves the step of inputting file name)
    • pXX_myo.nii to _myo.nii (Explanation: Same as above)
    • pXX_bp.nii to _bp.nii
  • Copy the ReplaceAtrialWall.bat file to the gc folder and double-click on it. This is the ReplaceAtrialWall code: 
magic -i _myo.nii -i2 _le.nii -o _atrial_wall.nii -t 1 --ms x 
del _myo.nii
  • Now run scar3d as: 
scar3d -w . x -m 3.0 -s 0.5 -auto x
  • Note if you are doing this for just processing ventricle images NOT from the challenge:
    • Run prescar3d _le.nii _gad_seg.nii (where _gad_seg.nii is a rough segmentation of bloodpool)
    • The _blood_pool.nii, _atrial_wall.nii generated from the above step needs to be replaced by appropriate blood pool and atrial wall (myocardium segmentation). These are the very steps described above
  • Keep an eye on the sink nodes, if it starts from 0 and goes upwards this means the iterative procedure is working towards a convergence. Divergence, i.e. sink nodes spiralling to a large value from the very beginning of the iteration is highly discouraged. In these circumstances, the -m and -s values should be readjusted

FWHM segmentation

  • Create a new folder called fwhm. Copy pXX_de.nii, pXX_myo.nii inside fwhm
  • Rename the files inside fwhm as follows:
    • pXX_de.nii to _le.nii (you are renaming this so that graph-cuts software knows what the DE-MRI scan is called, saves the step of inputting file name)
    • pXX_myo.nii to _myo.nii (Explanation: Same as above)
  • Open ITK-Snap and load greyscale image as _le.nii and draw a section of remote myocardium.
    • Note: it is not even important to get this right as FWHM core scar segmentation module does not use this. However, remote myocard segmentation is required for grey-zone segmentation. Grey-zone is not used as part of analysis here. Only core scar is necessary in subsequent steps. A mask for remote myocardium needs to be provided nevertheless
  • Run the batch file AutoFWHM by double-clicking, batch file commands here: 
mymask _le.nii _myo.nii _le_seg.nii 
mymask _le.nii _rm.nii _le_rm.nii 
vin -i _le.nii -m _le_seg.nii -r _le_rm.nii -o scar_seg_fw.nii -w . --fw x
magic2 -i1 scar_seg_fw.nii -o scar_seg_fw_core.nii -t1 2 -t2 0 --chbin x
  • This should generate a scar_seg_fw.nii and scar_seg_fw_core.nii. The former has both core scar and grey-zone. Latter only core scar.
  • As of 2013 there is now a FWHM developed primarly for atrium using user-selected seeded region-growing

Analysing FWHM with Graph-cuts

  • Copy scar_seg_fw_core.nii to gc fodler.
  • Run the AutoDice batch file by double-clicking: 
for %%f in (_gc_seg*) do (
echo %%f >> dice.txt
dice_2 scar_seg_fw_core.nii %%f 1 1 dice.txt 
)
  • This should generate a Dice.txt, open in text-editor and check which gc_seg_out generates the best dice. Say this is gc_seg_out_XX.nii
  • Type the following commands: 
threshold _gc_seg_out_XX.nii _gc_seg_out_final.nii 0.5
  • Now, to get the Dice numbers on final segmentation, type: 
Dice_2 scar_seg_fw_core.nii _gc_seg_out_final.nii 1 1
