Gap analysis

From Rashed Karim Wiki

Jump to: navigation, search

Details

Software for analysing gaps in scar. Use x to select points on atria shell, l to complete lines in roof and k to complete circles around veins.

See also

Software for computing encirclement


Code

Version 1.0

  • Code for measuring gaps in scar
  • Code for measuring gaps in voltage. This code has color scales reversed and scalar value capped at 1.0.

Version 2.0

Major fix: The intensity profiles along shortest path lines (in encirclement mode using 'k' or 'l') was not ordered properly. [Discussion below in appendix]

  • Gaps version 2

Version 2.0a

This version was specially customized for Henry. Color map is clamped to 0.1 (ln 620): 

mapper->SetScalarRange(0, 0.1);      // modified
mapper->SetScalarModeToUsePointData();    
mapper->ScalarVisibilityOn(); 
vtkSmartPointer<vtkLookupTable> lut = vtkSmartPointer<vtkLookupTable>::New(); 
lut->SetTableRange(0, 0.1);    // modified
lut->SetHueRange(0.166, 0.0);  // modified
  • Gaps version 2a


Appendix

Fix in v. 2.0

  • The issue was mainly the use of associative arrays in C++ (i.e. maps) to store point IDs. This was done with the purpose of preventing duplicate point IDs being stored. Iterating maps using an iterator returned elements in an order which is not correct. The order is generally determined by a comparator function. It wasn't clear why the comparator for vtkIdType was not ordering returned elements.
