The Problem

The surface mesh that can be easily represented as a undirected graph.
Geodesic distances computed on the map from the marked point (marked in blue). Color changes from green to yellow to red with increasing distance.

The voltage values are not defined at every point on a left atrium surface mesh. It is only defined at some of the mesh vertices. The problem is to interpolate the unknown scalar values (i.e. voltage). One intuitive way to interpolate is to compute the weighted average of the known scalars. This amounts to finding geodesic distances between vertices of a mesh. VTK unfortunately comes with no magic function that can interpolate scalars in this manner. Although it does now (version >5.5) come with a class that can compute geodesics. Also I posted a question regarding this on the VTK forum.

Shortest path algorithms

I needed the Single-source shortest path algorithms where the best-path from one specified vertex to every other vertex in the graph is computed. The two options:

  • Bellman-ford (takes care of negative edge weights, but a little expensive)
  • Dijkstra's algorithm (cannot handle negative edge weights, but runs in O(VlogV) time ONLY if it uses a priority Queue

Priority Queue

At each iteration of the algorithm, the queue must return the vertex with minimum distance. If the queue is implemented as a simple array, this will mean going through each element at least once and thus O(n) time. However, implementations using binomial or fibonacci heaps can do better: O(log n)

An excellent lecture on the Dijkstra's algorithm can be found here.

Implementations

I found a good working implementation of Dijkstra's algorithm that implements a priority queue here. However, it didn't compile straight out-of-the-box. I had to make minor modifications:

  • fheap.cpp: trees = new FHeapNode*[maxTrees]; instead of trees = new ( FHeapNode* )[maxTrees]; by removing the brackets
  • fheap.cpp: the use of log in maxTrees = 1 + (int)(1.44 * log((double)n)/log(2.0)); where n should be casted to double
