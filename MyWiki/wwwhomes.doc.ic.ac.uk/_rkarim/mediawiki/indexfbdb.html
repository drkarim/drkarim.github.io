Introduction

This follows the MVC framework, Data and visualization are kept separate. The actor is added to a PropCollection and the PropCOllection is drawn on every onDraw of the main-window. Thus any changes to the data and thus actors on the scene is immediately brought into effect by the 'View' code. For example, see OnDraw of vtkMFCRenderView 

void vtkMFCRenderView::OnDraw(CDC* pDC)

Examples

  • Can be found in any VTK example source: C:\PhD\software\vtk562_source\Examples\GUI\Win32\SampleMFC
  • There is a bare-bones example of MFC-VTK within: C:\PhD\software\virgin_MFC_VTK

Tweaks

Tweaks that were required to get the bare-bones application above from a project generated using the default MFC Single document interface wizard. The bare-bones example builds a MFC application that integrates a VTK window with basic data loading and viewing.

Changes to your View class

  • Must inherit from vtkMFCRenderView, so: 
class CVenus_EditorView :  public vtkMFCRenderView { .. }
  • Must copy the onUpdate function, add this using 'Class view' -> Your view class -> Right click properties -> In property view (Overrides icon/tab) -> OnUpdate function -> Add -> Copy code below: 
void CVenus_EditorView::OnUpdate(CView* pSender, LPARAM lHint, CObject* pHint)
{
 // TODO: Add your specialized code here and/or call the base class
  vtkPropCollection *propc;
  vtkProp *prop;
  // TODO: Add your specialized code here and/or call the base class
  // first remove any old actors
  this->Renderer->GetViewProps()->RemoveAllItems();
  propc = this->GetDocument()->GetViewProps();
  propc->InitTraversal();
  while (prop = propc->GetNextProp())
    {
    this->Renderer->AddViewProp(prop);
    this->Renderer->ResetCamera();
    }
  
  this->vtkMFCRenderView::OnUpdate(pSender, lHint, pHint);      
}
  • Must delete the OnDraw function in View class (this is generated by VS2005 automatically). This allows the OnDraw function of vtkMFCRenderView (i.e. parent) to be invoked. Use 'Class view' -> Your view class -> Right click properties -> In property view

(Overrides icon/tab) -> Delete function.

Changes to your document class

  • Must inherit from vtkMFCDocument, so: 
class CVenus_EditorDoc : public vtkMFCDocument { ... }
  • Adding the Reader, mapper and actor to the Venus_EditorDoc.h header file 
  vtkDataSetReader *Reader;
  vtkDataSetMapper *Mapper;
  vtkActor *Actor;


  • Initialization of these within the constructor: 

CVenus_EditorDoc::CVenus_EditorDoc()
{
	// TODO: add one-time construction code here
	this->Reader = vtkDataSetReader::New();
	  this->Mapper = vtkDataSetMapper::New();
	  this->Actor = vtkActor::New();
	  this->Actor->SetMapper(this->Mapper);
	  this->Actor->VisibilityOff();
	this->Props->AddItem(this->Actor);

}
  • The onFileOpen function, it is important that this function is added through 'Class view' -> Your Doc class -> Right click properties -> Events (thunder icon)
