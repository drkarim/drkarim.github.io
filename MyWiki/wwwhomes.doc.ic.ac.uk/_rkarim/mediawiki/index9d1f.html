Plotting 3D data using mesh grid surface

Assuming you loaded your data in a matrix called dmap which is a 3x3 matrix X = dmap(:,1), Y=dmap(:,2) and Z=c=dmap(:,3)

Images can be loaded easily using the load function and changing the current directory to where the file is located. Directory can be changed using the drop down file explorer on top

Grids

So we often want a regular spacing of the X and Y locations to make it smoother and not faceted. A regularly spaced data set is called a grid. It uses up more memory and thus is slower to manipulate because data is defined at every location. To convert the irregularly spaced data to regularly spaced we need to grid it. This requires a couple of steps. First you need to define the spacing and extent of the grid by making two vectors of the X and Y upper and lower limits at a given spacing: 

>> rangeY=floor(min(dmap(:,2))):.2:ceil(max(dmap(:,2)))

>>rangeX=floor(min(dmap(:,1))):.2:ceil(max(dmap(:,1)))

0.2 is the spacing I have chosen. I was a little obscure in choosing the "floor(minâ€¦" syntax. Try "min(dmap(:,2))" to see what that gives you and then try "floor(min(dmap(:,2)))" to see what that does. Hint: it is a way of controlling the rounding direction.

Now make matrices for each X and Y location: 

>>[X,Y]=meshgrid(rangeX,rangeY)

And then interpolate your Z values across this grid: 

>>Z=griddata(dmap(:,1),dmap(:,2),dmap(:,3),X,Y)

To see the result:

>>surf(X,Y,Z)

Plotting Gaussians on the same graph

Here we plot two gaussian distributions with (f_mean, f_std) and (b_mean, b_std)

Commands 

x=0:1:2000;

y1=1/(sqrt(2*pi*f_Std*f_Std))*exp(-0.5*(x-f_mean).^2/(f_Std));

y2=1/(sqrt(2*pi*f_Std*f_Std))*exp(-0.5*(x-f_mean).^2/(f_Std));

plot(x,y1,x,y2);

The above function is wrong. use this: 

function f = gauss_distribution(x, mu, s)
p1 = -.5 * ((x - mu)/s) .^ 2;
p2 = (s * sqrt(2*pi));
f = exp(p1) ./ p2;

This is already saved as an .m file in experiments/matlab

And this directory is included in the path of my matlab software.

Need to only invoke: y = gauss_distribution(x,m,s);

plot(x,y);

Plotting a Histogram from a file containing the values

Assuming that the values to be represented into the histogram are found in a text file. x denotes the x-axis or the bin intervals. hist bins the elements in A according to x.

Commands 

A=load('intensity.txt');
>> x=0:1000; 
>> n=hist(A,x);
>> bar(x,n);

If you wanted a line-graph of the histogram, if D is where the data is 

[y,x] = hist(D, 50)
plot(x,y)

Finding parameters for a Gaussian Mixture Model (GMM)

So I set about to understand how to model observed data using a mixture of Gaussians (a.k.a Gaussian Mixture Model GMM). This is common in image analysis, where a background model could comprise of several tissue classes. The definition of a GMM is relatively simple and defined in its wiki page. It is a linear (convex) combination of two or more gaussian distributions with associated means and variances. In other words, a weighted combination of several gaussians.

To determine the parameters of GMM for a given dataset, this amounts to finding the weights, means and variances of the Gaussians. Although the GMM can be solved for n-dimensional datasets, here I will document only the 1-dimensional case.

The EM-algorithm is a common technique to find these parameters. Here is an implementation in MATLAB that seems to work well. A nice way to test this algorithm is to generate random samples drawn from two or three Gaussians with known mean and variance. In MATLAB: 

X = zeros(600,1); 
X(1:200,:) = normrnd(0,1,200,1); 
X(201:400,:) = normrnd(3,2,200,1); 
X(401:600,:) = normrnd(20,3,200,1);

This draws data randomly from three normal distributions centered at 0, 3 and 20 with variances 1,2 and 3 respectively (i.e. the 1st and 2nd params of normrnd). It fills the 1D dataset X sequentially with this data from rows 1-200, 201-400 and 401-600 with respect to the three distributions. Later, a call to the EM-algorithm implementation using: 

[W,M,V,L] = EM_GM(X,3,[],[],1,[]);

Finds the relevant parameters of the GMM in W,M,V and L. For example, a plotting of the individual gaussians is also possible: 

x=-10:0.5:30;
y1=1/(sqrt(2*pi*V(:,:,1)*V(:,:,1)))*exp(-0.5*(x-M(1,1)).^2/(V(:,:,1)));
y2=1/(sqrt(2*pi*V(:,:,2)*V(:,:,2)))*exp(-0.5*(x-M(1,2)).^2/(V(:,:,2)));
y3=1/(sqrt(2*pi*V(:,:,3)*V(:,:,3)))*exp(-0.5*(x-M(1,3)).^2/(V(:,:,3)));
y4 = W(1,1)*y1+W(1,2)*y2+W(1,3)*y3;
plot(x,y1,x,y2,x,y3,x,y4);

This plot clearly shows how a weighting of the individual gaussians produces the GMM model in y4.

Loading NIFTI or ANALYZE images

  • The module that implements the nifti and analyze medical images can be found here. A very nice tool, once downloaded place all source files in a folder and point your MATLAB's path to that folder by going to File -> Set Path.
  • To load images, use commands like: 
nii = load_untouch_nii('p1nii.nii');

Note that I use the load_untouch command here as it ignores orientation, intensity scaling of images.

  • To now say plot a histogram, the image matrix will be stored in nii.img which will be an n-by-m array, of the n-by-m 2D image.
  • Convert the n-by-m array to nm-by-1 column vector: 
img = nii.img; 
imgcolvec = img(:);
x=0:2000; 
n=hist(imgcolvec, x); 
bar(x,n);

Plotting histograms of an image

Note we re-scale the histogram frequency (see below n_norm and n_2). n_norm normalizes the histogram between 0 and 1. n_2 re-scales the histogram to that of the Gaussian mixture curve. So the 0.02 factor is the maximum of the Gaussian mixture curve. Normally, this value can be determined by looking at the plot generated from the EM command: EM_GM(A,4,20,[],1,[]), which draws the individual and combined gaussians following the EM computation. 

A=load('_dilated_seg_hist.txt'); 
n=0:400; n=hist(A,x); n_norm=n/max(n);
[W,M,V,L] = EM_GM(A,4,20,[],1,[]);
n_2 = n_norm*0.02;
hold on
stairs(x,n_2);
